# OpenCode Configuration Guide, The best way to configure opencode

Welcome to the comprehensive configuration guide for OpenCode! This document covers all aspects of configuring your OpenCode environment for optimal performance and customization.

> **Note**: This is a comprehensive configuration guide that covers all major features and options available in OpenCode.

---

## Quick Start

To get started with OpenCode configuration, create a `opencode.json` file in your project root:

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "theme": "opencode",
  "model": "anthropic/claude-sonnet-4-20250514",
  "autoupdate": true
}
```

---

## Core Configuration Options

### Basic Settings

The most essential configuration options for OpenCode:

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `theme` | string | `"opencode"` | Visual theme for the interface |
| `model` | string | `"anthropic/claude-sonnet-4-20250514"` | Primary AI model to use |
| `autoupdate` | boolean | `true` | Enable automatic updates |

### Advanced Settings

For power users who need more control:

```jsonc title="opencode.jsonc"
{
  "$schema": "https://opencode.ai/config.json",
  "theme": "opencode",
  "model": "anthropic/claude-sonnet-4-20250514",
  "small_model": "anthropic/claude-3-5-haiku-20241022",
  "autoupdate": true,
  "share": "manual",
  "permission": {
    "edit": "ask",
    "bash": "ask"
  }
}
```

## Model Configuration

### Provider Setup

Configure different AI providers and their models:

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "anthropic": {
      "models": {
        "claude-sonnet-4": "anthropic/claude-sonnet-4-20250514",
        "claude-haiku": "anthropic/claude-3-5-haiku-20241022"
      },
      "options": {
        "apiKey": "{env:ANTHROPIC_API_KEY}"
      }
    },
    "openai": {
      "models": {
        "gpt-4": "openai/gpt-4",
        "gpt-3.5": "openai/gpt-3.5-turbo"
      },
      "options": {
        "apiKey": "{env:OPENAI_API_KEY}"
      }
    }
  }
}
```

### Model Selection

Choose the right model for different tasks:

- **Claude Sonnet 4**: Best for complex reasoning and code generation
- **Claude Haiku**: Fast and cost-effective for simple tasks
- **GPT-4**: Excellent for creative writing and analysis
- **GPT-3.5**: Good balance of speed and capability

## Agent Configuration

### Built-in Agents

OpenCode comes with several pre-configured agents:

```jsonc title="opencode.jsonc"
{
  "$schema": "https://opencode.ai/config.json",
  "agent": {
    "build": {

      }
    },
    "plan": {

      "model": "anthropic/claude-haiku-4-20250514",

        "write": false,
```

### Custom Agent Creation

Create your own specialized agents:

```markdown title="~/.config/opencode/agent/custom.md"
---
description: Custom agent for specific project needs
mode: subagent
model: anthropic/claude-sonnet-4-20250514
temperature: 0.1
tools:
  write: true
  edit: true
  bash: false
---

You are a custom agent specialized in [your specific domain].
Focus on [specific tasks or areas of expertise].
```

## Permission System

### Security Levels

Configure what AI agents can do in your environment:

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "permission": {
    "edit": "ask",
    "bash": {
      "*": "ask",
      "safe-commands": "allow"
    },
    "webfetch": "allow"
  }
}
```

### Permission Options

- **`"allow"`**: No confirmation required
- **`"ask"`**: User confirmation required
- **`"deny"`**: Operation blocked

## Formatter Configuration

### Built-in Formatters

Configure code formatting tools:

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "formatter": {
    "prettier": {
      "disabled": false,
      "config": ".prettierrc"
    },
    "eslint": {
      "disabled": false,
      "fix": true
    },
    "black": {
      "disabled": false,
      "line_length": 88
    }
  }
}
```

### Custom Formatters

Add your own formatting commands:

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "formatter": {
    "custom-prettier": {
      "command": ["npx", "prettier", "--write", "$FILE"],
      "environment": {
        "NODE_ENV": "development"
      },
      "extensions": [".js", ".ts", ".jsx", ".tsx"]
    }
  }
}
```

## MCP Server Configuration

### Model Context Protocol

Configure MCP servers for enhanced AI capabilities:

```json title="opencode.json"

        "args": ["-y", "@modelcontextprotocol/server-filesystem"]
```

## Environment Variables

### Variable Substitution

Use environment variables in your configuration:

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "model": "{env:OPENCODE_MODEL}",
  "provider": {
    "anthropic": {
      "options": {
        "apiKey": "{env:ANTHROPIC_API_KEY}"
      }
    }
  }
}
```

### File Content Substitution

Include file contents in your configuration:

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "instructions": [
    "{file:./CONTRIBUTING.md}",
    "{file:./docs/guidelines.md}"
  ]
}
```

## Best Practices

### Configuration Organization

1. **Global Settings**: Place common settings in `~/.config/opencode/opencode.json`
2. **Project Settings**: Override with project-specific `opencode.json`
3. **Environment Variables**: Use for sensitive data like API keys
4. **File References**: Include large instruction files without cluttering config

### Security Considerations

- Never commit API keys directly to version control
- Use environment variables for sensitive configuration
- Review permissions carefully before allowing AI operations
- Test configurations in safe environments first

## Troubleshooting

### Common Issues

| Problem | Solution |
|---------|----------|
| Configuration not loaded | Check file path and JSON syntax |
| Model not available | Verify provider configuration and API keys |
| Permissions too restrictive | Adjust permission levels in config |
| Theme not applied | Ensure theme file exists and is valid |

### Debug Mode

Enable debug logging:

```bash
export OPENCODE_DEBUG=1
opencode run "test command"
```
