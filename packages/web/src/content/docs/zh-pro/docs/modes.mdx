opencode 中的模式允许你为不同使用场景自定义行为、工具和提示内容。

它带有两个内置模式：**build** 和 **plan**。你可以自定义这些模式或通过 opencode 配置自己的模式。

你可在会话过程中切换模式，或在配置文件中进行设置。




---

## 内置模式

opencode 带有两种内置模式。

---

### 构建（Build）

Build 是**默认**模式，启用所有工具。这是开发工作的标准模式，在此模式下你需要完全访问文件操作和系统命令。

---

### 规划（Plan）

一个为规划和分析设计的限制模式。在规划模式中，以下工具默认被禁用：

- `write` - 无法创建新文件
- `edit` - 无法修改现有文件
- `patch` - 无法应用补丁
- `bash` - 无法执行 shell 命令

当你希望 AI 分析代码、提出修改建议或制定计划而不对你的代码库进行任何实际修改时，此模式很有用。

---

## 切换

你可以使用 _Tab_ 键在会话期间切换模式。也可使用已配置的 `switch_mode` 快捷键。

另见：[格式化器](/zh/docs/formatters)了解代码格式化配置相关信息。

## 配置

你可以自定义内置模式或通过配置创建自己的模式。模式有两种配置方式：

### JSON 配置

在 `opencode.json` 配置文件中配置模式：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "mode": {
    "build": {
      "model": "anthropic/claude-sonnet-4-20250514",
      "prompt": "{file:./prompts/build.txt}",
      "tools": {
        "write": true,
        "edit": true,
        "bash": true
      }
    },
    "plan": {
      "model": "anthropic/claude-haiku-4-20250514",
      "tools": {
        "write": false,
        "edit": false,
        "bash": false
      }
    }
  }
}
```

### Markdown 配置

你也可以使用 markdown 文件定义模式。将文件放置在以下位置：

- 全局：`~/.config/opencode/mode/`
- 项目：`.opencode/mode/`

以下是 ~/.config/opencode/mode/review.md 的示例：

```markdown title="~/.config/opencode/mode/review.md"
---
model: anthropic/claude-sonnet-4-20250514
temperature: 0.1
tools:
  write: false
  edit: false
  bash: false
---

你处于代码审查模式。重点关注：

- 代码质量和最佳实践
- 潜在 bug 和边界情况
- 性能影响
- 安全考虑

提供建设性反馈而不进行直接更改。
```

markdown 文件名会成为模式名称（例如，`review.md` 会创建一个 `review` 模式）。

下面详细看看这些配置选项。

---

### 模型

使用 `model` 配置来覆盖此模式的默认模型。这对于使用针对不同任务优化的不同模型很有用。例如，用更快的模型做规划，用更强大的模型来实施。

```json title="opencode.json"
{
  "mode": {
    "plan": {
      "model": "anthropic/claude-haiku-4-20250514"
    }
  }
}
```

---

### （温度）Temperature

使用 `temperature` 配置控制 AI 响应的随机性和创造性。数值越低，响应越聚集、越具确定性，数值越高，创造性和变异性越强。

```json title="opencode.json"
{
  "mode": {
    "plan": {
      "temperature": 0.1
    },
    "creative": {
      "temperature": 0.8
    }
  }
}
```

Temperature 值通常在 0.0 到 1.0 之间：

- **0.0-0.2**：响应高度聚集、确定，适合代码分析和规划
- **0.3-0.5**：平衡的响应，具有一定的创造性，适用于一般开发任务
- **0.6-1.0**：更具创造性和多样化的响应，适用于头脑风暴和探索

```json title="opencode.json"
{
  "mode": {
    "analyze": {
      "temperature": 0.1,
      "prompt": "{file:./prompts/analysis.txt}"
    },
    "build": {
      "temperature": 0.3
    },
    "brainstorm": {
      "temperature": 0.7,
      "prompt": "{file:./prompts/creative.txt}"
    }
  }
}
```

如果未指定 temperature，opencode 会使用模型特定的默认值（大多数模型通常为 0，Qwen 模型为 0.55）。

---

### 提示词

使用 `prompt` 配置为此模式指定自定义系统提示文件。提示文件应包含与模式用途相关的特定指令。

```json title="opencode.json"
{
  "mode": {
    "review": {
      "prompt": "{file:./prompts/code-review.txt}"
    }
  }
}
```

此路径相对于配置文件所在的位置。因此这适用于全局 opencode 配置和项目特定配置。

---

### 工具

使用 `tools` 配置控制此模式中可用的工具。你可以通过将它们设置为 `true` 或 `false` 来启用或禁用特定工具。

```json
{
  "mode": {
    "readonly": {
      "tools": {
        "write": false,
        "edit": false,
        "bash": false,
        "read": true,
        "grep": true,
        "glob": true
      }
    }
  }
}
```

如果未指定工具，默认启用所有工具。

---

#### 可用工具

以下是通过模式配置可以控制的所有工具。

| 工具        | 描述             |
| ----------- | ----------------------- |
| `bash`      | 执行 shell 命令  |
| `edit`      | 修改现有文件   |
| `write`     | 创建新文件        |
| `read`      | 读取文件内容      |
| `grep`      | 搜索文件内容    |
| `glob`      | 按模式查找文件   |
| `list`      | 列出目录内容 |
| `patch`     | 对文件应用补丁  |
| `todowrite` | 管理待办事项列表       |
| `todoread`  | 读取待办事项列表         |
| `webfetch`  | 获取网络内容       |

## 自定义模式

你可以通过将自定义模式添加到配置中来创建自己的模式。以下是使用两种方法的示例：

### 使用 JSON 配置

```json title="opencode.json" {4-14}
{
  "$schema": "https://opencode.ai/config.json",
  "mode": {
    "docs": {
      "prompt": "{file:./prompts/documentation.txt}",
      "tools": {
        "write": true,
        "edit": true,
        "bash": false,
        "read": true,
        "grep": true,
        "glob": true
      }
    }
  }
}
```

### 使用 markdown 文件

在 `.opencode/mode/` 中创建模式文件用于项目特定模式，或在 `~/.config/opencode/mode/` 中用于全局模式：

以下是 .opencode/mode/debug.md 的示例：

```markdown title=".opencode/mode/debug.md"
---
temperature: 0.1
tools:
  bash: true
  read: true
  grep: true
  write: false
  edit: false
---

你处于调试模式。你的主要目标是协助调查和诊断问题。

重点关注：

- 通过仔细分析理解问题
- 使用 bash 命令检查系统状态
- 读取相关文件和日志
- 搜索模式和异常情况
- 清晰阐释发现的问题

不要对文件进行任何更改。只进行调查和报告。
```

```markdown title="~/.config/opencode/mode/refactor.md"
---
model: anthropic/claude-sonnet-4-20250514
temperature: 0.2
tools:
  edit: true
  read: true
  grep: true
  glob: true
---

你处于重构模式。重点是在不改变功能的情况下提高代码质量。

优先级：

- 提高代码可读性和可维护性
- 统一命名规范
- 减少代码重复
- 合理优化性能
- 确保所有测试继续通过
```

### 使用场景

以下是不同模式的一些常见使用场景。

- **Build 模式**：启用所有工具，开展完整开发工作
- **Plan 模式**：仅分析和规划，不实际修改代码
- **Review 模式**：只读访问权限加文档工具，用于代码审查
- **Debug 模式**：启用 bash 和读取工具，专注问题排查
- **Docs 模式**：可进行文件操作，但没有系统命令，用于文档编写

你可能还会发现不同的模型适用于不同的使用场景。
