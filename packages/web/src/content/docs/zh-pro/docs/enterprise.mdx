opencode 不存储你的任何代码或上下文数据。这使你能轻松在所在组织中使用 opencode 。

要开始使用，我们建议：

1. 在团队内部进行试用。
2. [**联系我们**](mailto:hello@sst.dev)讨论定价和实施选项。

## 试用

由于 opencode 是开源的，不存储你的任何代码或上下文数据，你的开发人员可以直接[开始使用](/zh/docs/)并进行试用。

---

### 数据处理

**opencode 不存储你的代码或上下文数据。** 所有处理都在本地进行，或通过直接调用你的AI提供商的API完成。

这里唯一需要注意的是可选的 `/share` 功能。

---

#### 分享对话

如果用户启用 `/share` 功能，对话和与之相关的数据会发送到我们用于在 opencode.ai 托管分享页面的服务。

当前这些数据通过我们的 CDN 边缘网络提供服务，并缓存在靠近用户的边缘节点。

我们建议你在试用期间禁用此功能。

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "share": "disabled"
}
```

[了解更多关于分享的信息](/zh/docs/share)。

---

### 代码所有权

**你拥有 opencode 生成的所有代码。** 没有许可限制或所有权声明。

## 部署

一旦你完成试用并准备在你的组织内自托管 opencode，你可以[**联系我们**](mailto:hello@sst.dev)讨论定价和实施选项。

### 单点登录

试用结束后，可为企业部署实施单点登录（SSO）集成。
这将会让你团队的会话数据和共享对话受到企业认证系统的保护。

---

### 私有 NPM

opencode 通过 Bun 原生的 `.npmrc` 文件支持，可对接私有 npm 注册表。如果你的组织使用私有注册表，如 JFrog Artifactory、Nexus 或类似产品，确保开发人员在运行 opencode 之前已通过认证。

要为私有注册表配置身份认证：

```bash
npm login --registry=https://your-company.jfrog.io/api/npm/npm-virtual/
```

此操作会创建包含身份认证详情的 `~/.npmrc`文件。opencode 会自动识别这些文件。

:::警告
在运行 opencode 之前，你必须登录到私有注册表。
:::

或者，你可以手动配置 `.npmrc` 文件：

```bash title="~/.npmrc"
registry=https://your-company.jfrog.io/api/npm/npm-virtual/
//your-company.jfrog.io/api/npm/npm-virtual/:_authToken=${NPM_AUTH_TOKEN}
```

开发人员必须在运行 opencode 之前登录到私有注册表，以确保可以从企业注册表安装软件包。

---

### 自托管

分享功能可以自托管，且分享页面仅在用户通过认证后才可访问。
