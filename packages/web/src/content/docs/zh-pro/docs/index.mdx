让我们开始吧。

#### 前置条件

要使用 opencode，你需要：

1. 一个现代化的终端模拟器，比如：

   - [WezTerm](https://wezterm.org)，跨平台
   - [Alacritty](https://alacritty.org)，跨平台
   - [Ghostty](https://ghostty.org)，适用于Linux 和 macOS
   - [Kitty](https://sw.kovidgoyal.net/kitty/)，适用于Linux 和 macOS

2. 您想要使用的 LLM 提供商的 API 密钥。

---

## 安装

安装 opencode 最简单的方法是通过安装脚本。

```bash
curl -fsSL https://opencode.ai/install | bash
```

您也可以通过以下方式安装：

- **使用 Node.js**

  <Tabs>
    <TabItem label="npm">
      ```bash
      npm install -g opencode-ai
      ```
    </TabItem>
    <TabItem label="Bun">
      ```bash
      bun install -g opencode-ai
      ```
    </TabItem>
    <TabItem label="pnpm">
      ```bash
      pnpm install -g opencode-ai
      ```
    </TabItem>
    <TabItem label="Yarn">
      ```bash
      yarn global add opencode-ai
      ```
    </TabItem>
  </Tabs>

- **在 macOS 上使用 Homebrew**

  ```bash
  brew install sst/tap/opencode
  ```

- **在 Arch Linux 上使用 Paru**

  ```bash
  paru -S opencode-bin
  ```

#### Windows系统

目前，自动安装方法在 Windows 系统上无法正常运行。不过，您可以从 [Releases](https://github.com/sst/opencode/releases) 下载二进制文件。

## 配置

使用 opencode时，您可以通过配置 API 密钥来使用任意 LLM 提供商的用血。

我们建议注册 [Claude Pro](https://www.anthropic.com/news/claude-pro) 或 [Max](https://www.anthropic.com/max)，这是使用 opencode 最具性价比的方式。

注册完成后，运行 `opencode auth login` 并选择 Anthropic。

```bash
$ opencode auth login

┌  Add credential
│
◆  Select provider
│  ● Anthropic (recommended)
│  ○ OpenAI
│  ○ Google
│  ○ Amazon Bedrock
│  ○ Azure
│  ○ DeepSeek
│  ○ Groq
│  ...
└
```

或者，您可以选择其他提供商中的任意一个。[了解更多](/zh/docs/providers#directory)。

## 初始化

现在您已经配置好了提供商，接下来可以导航到您想要处理的项目所在位置。

```bash
cd /path/to/project
```

然后运行 opencode。

```bash
opencode
```

接下来，通过运行以下命令为项目初始化 opencode。

```bash frame="none"
/init
```

这会让 opencode 分析您的项目，并在项目根目录创建一个 `AGENTS.md` 文件。

:::tip
您应该将项目的 `AGENTS.md` 文件提交到 Git。
:::

这有助于 opencode 理解项目结构以及所使用的编码模式。

## 使用

现在您已经准备好使用 opencode 来处理您的项目了。尽管向它提问吧！

如果您是第一次使用 AI 编程智能体，这里有一些示例可能会对您有所帮助。

---

### 提问

可以让 opencode 为您解释代码库。

```txt frame="none"
How is authentication handled in @packages/functions/src/api/index.ts
```

如果您对代码库中自己未曾参与开发的部分不太了解，这种方式会很有帮助。

:::tip
使用 `@` 键可以在项目中模糊搜索文件。
:::

---

### 添加功能

您可以让 opencode 为您的项目添加新功能。不过，我们首先建议您让它创建一个方案。

1. **创建方案**

   opencode 有一个_方案模式_，在此模式下，它会禁用实际修改，而是建议_如何_实现功能。

   您可以使用 **Tab** 键切换到该模式。在界面右下角会显示相应的模式指示标识。

   ```bash frame="none" title="切换到计划模式"
   <TAB>
   ```

   现在，描述一下您希望它实现的功能。

   ```txt frame="none"
   When a user deletes a note, we'd like to flag it as deleted in the database.
   Then create a screen that shows all the recently deleted notes.
   From this screen, the user can undelete a note or permanently delete it.
   ```

   您需要向 opencode 提供足够详细的信息，让它来理解您的需求。您可以把它当作团队里的初级开发人员来沟通。

   :::tip
   给 opencode 充足的背景信息和示例，帮助它理解您的需求。
   :::

2. **迭代方案**

   它为您生成一个方案，您可以给出反馈或补充更多细节。

   ```txt frame="none"
   We'd like to design this new screen using a design I've used before.
   [Image #1] Take a look at this image and use it as a reference.
   ```

   :::tip
   将图片拖放到终端中，将其添加到提示内容中。
   :::

   opencode 可以扫描您提供的任何图片并将其添加到提示中。您可以通过将图片拖放到终端中来实现这一点。

3. **构建功能**

   如果您对方案感到满意，再次按 **Tab** 键切换回_构建模式_。

   ```bash frame="none"
   <TAB>
   ```

   并要求它进行修改。

   ```bash frame="none"
   Sounds good! Go ahead and make the changes.
   ```

---

### 进行修改

对于更简单直接的修改，您可以让 opencode 直接构建，而不必先审核方案。

```txt frame="none"
We need to add authentication to the /settings route. Take a look at how this is
handled in the /notes route in @packages/functions/src/notes.ts and implement
the same logic in @packages/functions/src/settings.ts
```

确保提供足够详细的信息，这样 opencode 才能做出正确的修改。

## 分享

您与 opencode 的对话可以[与您的团队分享](/zh/docs/share)。

```bash frame="none"
/share
```

这会创建一个指向当前对话的链接，并将其复制到剪贴板。

:::note
对话默认不会共享。
:::

这里有一个与 opencode 的[示例对话](https://opencode.ai/s/4XP1fce5)。
