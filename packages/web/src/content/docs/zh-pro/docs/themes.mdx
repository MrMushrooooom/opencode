使用 opencode 时，你可以从多个内置主题中选择一个，使用适配你终端主题的主题，或定义你自己的自定义主题。

默认情况下，opencode 使用我们自己的 `opencode` 主题。

---

## 终端要求

要使主题完整显示其调色板，你的终端必须支持 **truecolor**（24 位颜色）。大多数现代终端默认支持此功能，但你可能需要启用它：

- **检查支持情况**：运行 `echo $COLORTERM` - 输出应为 `truecolor` 或 `24bit`
- **启用 truecolor**：在 shell 配置文件中设置环境变量 `COLORTERM=truecolor`
- **终端兼容性**：确保你的终端模拟器支持 24 位颜色（大多数现代终端如 iTerm2、Alacritty、Kitty、Windows Terminal 和 GNOME Terminal 的较新版本都支持）

如果没有 truecolor 支持，主题颜色显示可能不准确，或回退到最接近的 256 色近似值。

---

## 内置主题

opencode 带有几个内置主题。

| 名称         | 描述                                |
| ------------ | ------------------------------------------ |
| `system`     | 适应你终端的背景颜色 |
| `tokyonight` | 基于 Tokyonight 主题              |
| `everforest` | 基于 Everforest 主题              |
| `ayu`        | 基于 Ayu 深色主题                |
| `catppuccin` | 基于 Catppuccin 主题              |
| `gruvbox`    | 基于 Gruvbox 主题                 |
| `kanagawa`   | 基于 Kanagawa 主题                |
| `nord`       | 基于 Nord 主题                    |
| `matrix`     | 黑客风格的绿黑主题          |
| `one-dark`   | 基于 Atom One Dark 主题           |

我们还在不断添加新主题。

---

## 系统主题

`system` 主题可自动适配你终端的配色方案。与使用固定颜色的传统主题不同，_system_ 主题具备以下特点：

- **生成灰度**：基于你终端的背景颜色创建自定义灰度，确保最佳对比度。
- **使用 ANSI 颜色**：利用标准 ANSI 颜色（0-15）进行语法高亮和 UI 元素渲染，这些颜色尊重你终端的调色板。
- **保留终端默认值**：对文本和背景颜色使用 `none` 以保持你终端的原生外观。

系统主题适用于以下用户：

- 希望 opencode 匹配其终端外观
- 使用自定义终端配色方案
- 偏好所有终端应用保持一致的外观

---

## 使用主题

你可以通过 `/theme` 命令调出主题选择器来选择主题。或者你可以在[配置](/zh/docs/config)中指定主题。

```json title="opencode.json" {3}
{
  "$schema": "https://opencode.ai/config.json",
  "theme": "tokyonight"
}
```

## 自定义主题

opencode 支持灵活的基于 JSON 的主题系统，允许用户轻松创建和自定义主题。

---

### 层次结构

主题按以下顺序从多个目录加载，后面的目录会覆盖前面的目录：

1. **内置主题** - 这些嵌入在二进制文件中
2. **用户配置目录** - 在 `~/.config/opencode/themes/*.json` 或 `$XDG_CONFIG_HOME/opencode/themes/*.json` 中定义
3. **项目根目录** - 在 `<project-root>/.opencode/themes/*.json` 中定义
4. **当前工作目录** - 在 `./.opencode/themes/*.json` 中定义

如果多个目录存在同名主题，将使用优先级更高目录中的主题。

---

### 创建主题

要创建自定义主题，需在任意主题目录中新建一个 JSON 文件。

对于用户级主题：

```bash no-frame
mkdir -p ~/.config/opencode/themes
vim ~/.config/opencode/themes/my-theme.json
```

对于项目特定主题：

```bash no-frame
mkdir -p .opencode/themes
vim .opencode/themes/my-theme.json
```

---

### JSON 格式

主题使用灵活的 JSON 格式，支持以下内容：

- **十六进制颜色**：`"#ffffff"`
- **ANSI 颜色**：`3`（0-255）
- **颜色引用**：`"primary"` 或自定义
- **深色/浅色变体**：`{"dark": "#000", "light": "#fff"}`
- **无颜色**：`"none"` - 使用终端的默认颜色或透明

---

### 颜色定义

`defs` 部分是可选的，可用于定义能在主题中复用的颜色。

---

### 终端默认值

特殊值 `"none"` 可用于任何颜色配置以继承终端的默认颜色。这对于创建与你终端配色方案无缝融合的主题特别有用：

- `"text": "none"` - 使用终端的默认前景色
- `"background": "none"` - 使用终端的默认背景色
