---
title: 分享
description: 分享你的 opencode 对话。
---

opencode 的分享功能允许你为 opencode 对话创建公共链接，这样你就可以与团队成员协作或向他人寻求帮助。

:::注意
共享的对话对任何有链接的人都是公开可访问的。
:::

---

## 工作原理

当你分享对话时，opencode会：

1. 为你的会话创建唯一的公共 URL
2. 将你的对话历史同步到我们的服务器
3. 使对话可通过共享链接访问 — `opencode.ai/s/<share-id>`

---

## 分享

opencode 支持三种分享模式，用于控制对话的共享方式：

---

### 手动（默认）

默认情况下，opencode 使用手动分享模式。会话不会自动共享，但你可以使用 `/share` 命令手动分享：

```
/share
```

这将生成一个唯一的 URL，并复制到你的剪贴板。

要在[配置文件](/zh/docs/config)中明确设置手动模式，可按如下方式：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "share": "manual"
}
```

---

### 自动分享

你可以通过在[配置文件](/zh/docs/config)中将 `share` 选项设置为 `"auto"` 来为所有新对话启用自动分享功能，配置方式如下：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "share": "auto"
}
```

启用自动分享后，每个新对话都会自动分享并生成一个链接。

---

### 禁用

你可以通过在[配置文件](/zh/docs/config)中将 `share` 选项设置为 `"disabled"` 来完全禁用分享功能：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "share": "disabled"
}
```

要在团队中为特定项目强制执行此操作，请将上述配置添加到项目的 `opencode.json` 中并提交到 Git。

---

## 取消分享

要停止分享对话并从公共访问中删除，可执行以下操作：

```
/unshare
```

这将删除分享链接并删除与对话相关的数据。

---

## 隐私

分享对话时有几点需要留意。

---

### 数据留存

共享的对话在你明确取消分享之前始终可访问。这包括：

- 完整的对话历史
- 所有消息和回复
- 会话元数据

---

### 建议

- 只分享不包含敏感信息的对话。
- 在分享前审查对话内容。
- 协作完成后取消分享对话。
- 避免分享包含专有代码或机密数据的对话。
- 对于敏感项目，完全禁用分享功能。

---

## 企业版

对于企业部署，分享功能可以按以下方式设置：

- **完全禁用**以符合安全合规要求
- **限制**为仅通过 SSO 认证的用户
- **自托管**在企业自有基础设施上

[了解更多](/zh/docs/enterprise)关于在企业组织中使用 opencode 的信息。
