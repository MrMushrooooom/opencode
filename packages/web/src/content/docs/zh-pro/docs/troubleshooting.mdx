要调试 opencode 的任何问题，你可以检查日志或它本地存储的会话数据。

---

### 日志

日志文件写入到：

- **macOS/Linux**：`~/.local/share/opencode/log/`
- **Windows**：`%APPDATA%\opencode\log\`

日志文件以时间戳命名（例如，`2025-01-09T123456.log`），并保留最近的 10 个日志文件。

你可以在[配置文件](/zh/docs/config#logging)中配置日志级别以获得更详细的调试信息。

---

### 存储

opencode 在磁盘上存储会话数据和其他应用程序数据：

- **macOS/Linux**：`~/.local/share/opencode/`
- **Windows**：`%USERPROFILE%\.local\share\opencode`

此目录包含：

- `auth.json` - 认证数据，如 API 密钥、OAuth 令牌
- `log/` - 应用程序日志
- `project/` - 项目特定数据，如会话和消息数据
  - 如果项目在 Git 仓库内，它存储在 `./<project-slug>/storage/` 中
  - 如果不是 Git 仓库，它存储在 `./global/storage/` 中

## 常见问题

以下是一些常见问题及其解决方法。

---

### opencode 无法启动

1. 检查日志中的错误消息
2. 尝试使用 `--print-logs` 运行以在终端中查看输出
3. 使用 `opencode upgrade` 确保你有最新版本

---

### 认证问题

1. 尝试使用 `opencode auth login <provider>` 重新认证
2. 检查你的 API 密钥是否有效
3. 确保你的网络允许连接到提供商的 API

---

### 模型不可用

1. 检查你是否已与提供商认证
2. 验证配置中的模型名称是否正确
3. 某些模型可能需要特定的访问权限或订阅

---

### Linux 上的复制/粘贴不工作

Linux 用户需要安装以下剪贴板工具之一才能使复制/粘贴功能正常工作：

**对于 X11 系统：**

```bash
apt install -y xclip
# 或
apt install -y xsel
```

**对于 Wayland 系统：**

```bash
apt install -y wl-clipboard
```

**对于无头环境：**

```bash
apt install -y xvfb
# 并运行：
Xvfb :99 -screen 0 1024x768x24 > /dev/null 2>&1 &
export DISPLAY=:99.0
```

opencode 会检测你是否使用 Wayland 并优先使用 `wl-clipboard`，否则它会按以下顺序尝试查找剪贴板工具：`xclip` 和 `xsel`。
