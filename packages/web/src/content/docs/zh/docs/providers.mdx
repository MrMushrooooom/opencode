---
title: 提供商
description: 在 opencode 中使用任何 LLM 提供商。
---

opencode 使用 [AI SDK](https://ai-sdk.dev/) 和 [Models.dev](https://models.dev) 来支持 **75+ LLM 提供商**，并且支持运行本地模型。

要添加提供商，您需要：

1. 使用 `opencode auth login` 添加提供商的 API 密钥。
2. 在您的 opencode 配置中配置提供商。

---

### 凭据

当您使用 `opencode auth login` 添加提供商的 API 密钥时，它们存储在 `~/.local/share/opencode/auth.json` 中。

---

### 配置

您可以通过 opencode 配置中的 `provider` 部分自定义提供商。

---

#### 基础 URL

您可以通过设置 `baseURL` 选项自定义任何提供商的基础 URL。这在代理服务或自定义端点时很有用。

```json title="opencode.json" {6}
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "anthropic": {
      "options": {
        "baseURL": "https://api.anthropic.com/v1"
      }
    }
  }
}
```

## 自定义提供商

要添加未在 `opencode auth login` 中列出的任何**OpenAI 兼容**提供商：

:::tip
您可以在 opencode 中使用任何 OpenAI 兼容的提供商。大多数现代 AI 提供商都提供 OpenAI 兼容的 API。
:::

1. 运行 `opencode auth login` 并向下滚动到**其他**。

   ```bash
   $ opencode auth login

   ┌  Add credential
   │
   ◆  Select provider
   │  ...
   │  ● Other
   └
   ```

2. 为提供商输入唯一 ID。

   ```bash
   $ opencode auth login

   ┌  Add credential
   │
   ◇  Enter provider id
   │  myprovider
   └
   ```

   :::note
   选择一个难忘的 ID，您将在配置文件中使用它。
   :::

3. 输入提供商的 API 密钥。

   ```bash
   $ opencode auth login

   ┌  Add credential
   │
   ▲  This only stores a credential for myprovider - you will need configure it in opencode.json, check the docs for examples.
   │
   ◇  Enter your API key
   │  sk-...
   └
   ```

4. 在项目目录中创建或更新您的 `opencode.json` 文件：

   ```json title="opencode.json" ""myprovider"" {5-15}
   {
     "$schema": "https://opencode.ai/config.json",
     "provider": {
       "myprovider": {
         "npm": "@ai-sdk/openai-compatible",
         "name": "My AI ProviderDisplay Name",
         "options": {
           "baseURL": "https://api.myprovider.com/v1"
         },
         "models": {
           "my-model-name": {
             "name": "My Model Display Name"
           }
         }
       }
     }
   }
   ```

   以下是配置选项：

   - **npm**：要使用的 AI SDK 包，`@ai-sdk/openai-compatible` 用于 OpenAI 兼容的提供商
   - **name**：UI 中的显示名称。
   - **models**：可用模型。
   - **options.baseURL**：API 端点 URL。
   - **options.apiKey**：可选地设置 API 密钥，如果不使用 auth。
   - **options.headers**：可选地设置自定义头。

   下面示例中的更多高级选项。

5. 运行 `/models` 命令，您的自定义提供商和模型将出现在选择列表中。

---

##### 示例

这是一个设置 `apiKey` 和 `headers` 选项的示例。

```json title="opencode.json" {9,11}
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "myprovider": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "My AI ProviderDisplay Name",
      "options": {
        "baseURL": "https://api.myprovider.com/v1",
        "apiKey": "{env:ANTHROPIC_API_KEY}",
        "headers": {
          "Authorization": "Bearer custom-token"
        }
      },
      "models": {
        "my-model-name": {
          "name": "My Model Display Name"
        }
      }
    }
  }
}
```

我们使用 `env` 变量语法设置 `apiKey`，[了解更多](/docs/config#env-vars)。

### Amazon Bedrock

要在 opencode 中使用 Amazon Bedrock：

1. 前往 Amazon Bedrock 控制台中的**模型目录**，并请求访问您想要的模型。

   :::tip
   您需要在 Amazon Bedrock 中访问您想要的模型。
   :::

2. 您需要设置以下环境变量之一：

   - `AWS_ACCESS_KEY_ID`：您可以通过创建 IAM 用户并为其生成访问密钥来获得此密钥。
   - `AWS_PROFILE`：首先使用 `aws sso login` 通过 AWS IAM Identity Center（或 AWS SSO）登录。然后获取您想要使用的配置文件的名称。
   - `AWS_BEARER_TOKEN_BEDROCK`：您可以从 Amazon Bedrock 控制台生成长期 API 密钥。

   一旦您有了上述之一，在运行 opencode 时设置它。

   ```bash
   AWS_ACCESS_KEY_ID=XXX opencode
   ```

   或将其添加到项目根目录的 `.env` 文件中。

   ```bash title=".env"
   AWS_ACCESS_KEY_ID=XXX
   ```

   或将其添加到您的 bash 配置文件中。

   ```bash title="~/.bash_profile"
   export AWS_ACCESS_KEY_ID=XXX
   ```

3. 运行 `/models` 命令选择您想要的模型。

### Anthropic

我们建议注册 [Claude Pro](https://www.anthropic.com/news/claude-pro) 或 [Max](https://www.anthropic.com/max)，这是使用 opencode 最具成本效益的方式。

注册后，运行 `opencode auth login` 并选择 Anthropic。

```bash
$ opencode auth login

┌  Add credential
│
◆  Select provider
│  ● Anthropic (recommended)
│  ○ OpenAI
│  ○ Google
│  ...
└
```

在这里您可以选择**Claude Pro/Max**选项，它会打开您的浏览器并要求您进行身份验证。

```bash
$ opencode auth login
┌  Add credential
│
◇  Select provider
│  Anthropic
│
◆  Login method
│  ● Claude Pro/Max
│  ○ Create API Key
│  ○ Manually enter API Key
└
```

现在当您使用 `/models` 命令时，所有 Anthropic 模型都应该可用。

##### 使用 API 密钥

如果您没有 Pro/Max 订阅，您也可以选择**Create API Key**。它也会打开您的浏览器并要求您登录 Anthropic，并给您一个可以在终端中粘贴的代码。

或者如果您已经有 API 密钥，您可以选择**Manually enter API Key**并在终端中粘贴它。

### Azure OpenAI

1. 前往 [Azure 门户](https://portal.azure.com/) 并创建**Azure OpenAI**资源。您需要：

   - **资源名称**：这成为您 API 端点的一部分（`https://RESOURCE_NAME.openai.azure.com/`）
   - **API 密钥**：来自您资源的 `KEY 1` 或 `KEY 2`

2. 前往 [Azure AI Foundry](https://ai.azure.com/) 并部署模型。

   :::note
   部署名称必须与模型名称匹配，opencode 才能正常工作。
   :::

3. 运行 `opencode auth login` 并选择**Azure**。

   ```bash
   $ opencode auth login

   ┌  Add credential
   │
   ◆  Select provider
   │  ● Azure
   │  ...
   └
   ```

4. 输入您的 API 密钥。

   ```bash
   $ opencode auth login

   ┌  Add credential
   │
   ◇  Select provider
   │  Azure
   │
   ◇  Enter your API key
   │  _
   └
   ```

5. 将您的资源名称设置为环境变量：

   ```bash
   AZURE_RESOURCE_NAME=XXX opencode
   ```

   或将其添加到项目根目录的 `.env` 文件中：

   ```bash title=".env"
   AZURE_RESOURCE_NAME=XXX
   ```

   或将其添加到您的 bash 配置文件中：

   ```bash title="~/.bash_profile"
   export AZURE_RESOURCE_NAME=XXX
   ```

6. 运行 `/models` 命令选择您部署的模型。

### LM Studio

您可以将 opencode 配置为通过 LM Studio 使用本地模型。

```json title="opencode.json" "lmstudio" {5, 6, 8, 10-14}
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "lmstudio": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "LM Studio (local)",
      "options": {
        "baseURL": "http://127.0.0.1:1234/v1"
      },
      "models": {
        "google/gemma-3n-e4b": {
          "name": "Gemma 3n-e4b (local)"
        }
      }
    }
  }
}
```

在此示例中：

- `lmstudio` 是自定义提供商 ID。这可以是您想要的任何字符串。
- `npm` 指定要用于此提供商的包。这里，`@ai-sdk/openai-compatible` 用于任何 OpenAI 兼容的 API。
- `name` 是 UI 中提供商的显示名称。
- `options.baseURL` 是本地服务器的端点。
- `models` 是模型 ID 到其配置的映射。模型名称将显示在模型选择列表中。

## 故障排除

如果您在配置提供商时遇到问题，请检查以下内容：

1. **检查 auth 设置**：运行 `opencode auth list` 查看提供商的凭据是否添加到您的配置中。

   这不适用于依赖环境变量进行身份验证的提供商，如 Amazon Bedrock。

2. 对于自定义提供商，检查 opencode 配置并：

   - 确保在 `opencode auth login` 中使用的提供商 ID 与 opencode 配置中的 ID 匹配。
   - 为提供商使用正确的 npm 包。例如，为 Cerebras 使用 `@ai-sdk/cerebras`。对于所有其他 OpenAI 兼容的提供商，使用 `@ai-sdk/openai-compatible`。
   - 检查 `options.baseURL` 字段中使用的正确 API 端点。
