使用 opencode，您可以从几个内置主题中选择一个，使用适应您终端主题的主题，或者定义您自己的自定义主题。

默认情况下，opencode 使用我们自己的 `opencode` 主题。

---

## 终端要求

为了让主题以其完整的色彩调色板正确显示，您的终端必须支持 **真彩色**（24 位色彩）。大多数现代终端默认支持这一功能，但您可能需要启用它：

- **检查支持**：运行 `echo $COLORTERM` - 它应该输出 `truecolor` 或 `24bit`
- **启用真彩色**：在您的 shell 配置文件中设置环境变量 `COLORTERM=truecolor`
- **终端兼容性**：确保您的终端模拟器支持 24 位色彩（大多数现代终端如 iTerm2、Alacritty、Kitty、Windows Terminal 和最新版本的 GNOME Terminal 都支持）

没有真彩色支持，主题可能会显示出降低的色彩准确性或回退到最接近的 256 色近似值。

## 内置主题

opencode 带有几个内置主题。

| 名称         | 描述                    |
| ------------ | ----------------------- |
| `system`     | 适应您终端的背景颜色    |
| `tokyonight` | 基于 Tokyonight 主题    |
| `everforest` | 基于 Everforest 主题    |
| `ayu`        | 基于 Ayu 暗色主题       |
| `catppuccin` | 基于 Catppuccin 主题    |
| `gruvbox`    | 基于 Gruvbox 主题       |
| `kanagawa`   | 基于 Kanagawa 主题      |
| `nord`       | 基于 Nord 主题          |
| `matrix`     | 黑客风格的绿色黑色主题  |
| `one-dark`   | 基于 Atom One Dark 主题 |

还有更多，我们正在不断添加新主题。

## 系统主题

`system` 主题旨在自动适应您终端的配色方案。与使用固定颜色的传统主题不同，_system_ 主题：

- **生成灰度**：基于您终端的背景颜色创建自定义灰度，确保最佳对比度。
- **使用 ANSI 颜色**：利用标准 ANSI 颜色（0-15）进行语法高亮和 UI 元素，这些颜色会尊重您终端的调色板。
- **保持终端默认值**：对文本和背景颜色使用 `none` 以保持您终端的原生外观。

系统主题适合以下用户：

- 希望 opencode 匹配其终端外观
- 使用自定义终端配色方案
- 更喜欢所有终端应用程序的一致外观

---

## 使用主题

您可以通过使用 `/theme` 命令调出主题选择来选择主题。或者您可以在您的[配置](/docs/config)中指定它。

```json title="opencode.json" {3}
{
  "$schema": "https://opencode.ai/config.json",
  "theme": "tokyonight"
}
```

## 自定义主题

opencode 支持灵活的基于 JSON 的主题系统，允许用户轻松创建和自定义主题。

---

### 层次结构

主题从多个目录按以下顺序加载，后面的目录会覆盖前面的：

1. **内置主题** - 这些嵌入在二进制文件中
2. **用户配置目录** - 定义在 `~/.config/opencode/themes/*.json` 或 `$XDG_CONFIG_HOME/opencode/themes/*.json` 中
3. **项目根目录** - 定义在 `<project-root>/.opencode/themes/*.json` 中
4. **当前工作目录** - 定义在 `./.opencode/themes/*.json` 中

如果多个目录包含同名主题，将使用优先级更高的目录中的主题。

---

### 创建主题

要创建自定义主题，请在其中一个主题目录中创建 JSON 文件。

对于用户范围的主题：

```bash no-frame
mkdir -p ~/.config/opencode/themes
vim ~/.config/opencode/themes/my-theme.json
```

对于项目特定的主题。

```bash no-frame
mkdir -p .opencode/themes
vim .opencode/themes/my-theme.json
```

---

### JSON 格式

主题使用灵活的 JSON 格式，支持：

- **十六进制颜色**：`"#ffffff"`
- **ANSI 颜色**：`3`（0-255）
- **颜色引用**：`"primary"` 或自定义定义
- **深色/浅色变体**：`{"dark": "#000", "light": "#fff"}`
- **无颜色**：`"none"` - 使用终端的默认颜色或透明

---

### 颜色定义

`defs` 部分是可选的，它允许您定义可在主题中引用的可重用颜色。

---

### 终端默认值

特殊值 `"none"` 可用于任何颜色以继承终端的默认颜色。这对于创建与您终端配色方案无缝融合的主题特别有用：

- `"text": "none"` - 使用终端的默认前景色
- `"background": "none"` - 使用终端的默认背景色
