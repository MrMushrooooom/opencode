opencode 使用 [AI SDK](https://ai-sdk.dev/) 和 [Models.dev](https://models.dev) 来支持 **75+ 个 LLM 提供商**，并且支持运行本地模型。

---

## 提供商

最受欢迎的提供商默认预加载。如果你通过 `opencode auth login` 添加了提供商的凭据，当你启动 opencode 时它们将可用。

了解更多关于[提供商](/zh/docs/providers)。

---

## 选择模型

一旦你配置了提供商，你可以通过输入以下命令选择你想要的模型：

```bash frame="none"
/models
```

## 设置默认模型

要将其中一个设置为默认模型，你可以在 opencode 配置中设置 `model` 键。

```json title="opencode.json" {3}
{
  "$schema": "https://opencode.ai/config.json",
  "model": "lmstudio/google/gemma-3n-e4b"
}
```

这里的完整 ID 是 `provider_id/model_id`。

如果你配置了[自定义提供商](/zh/docs/providers#custom)，`provider_id` 是配置 `provider` 部分中的键，`model_id` 是 `provider.models` 中的键。

---

## 配置模型

你可以通过配置全局配置模型的选项。

```jsonc title="opencode.jsonc" {7-11}
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "openai": {
      "models": {
        "gpt-5": {
          "options": {
            "reasoningEffort": "high",
            "textVerbosity": "low",
            "reasoningSummary": "auto",
            "include": ["reasoning.encrypted_content"]
          }
        }
      }
    }
  }
}
```

这里我们为通过 `openai` 提供商使用的 `gpt-5` 模型设置全局选项。

你也可以为你使用的任何代理配置这些选项。代理配置会覆盖这里的任何全局选项。[了解更多](/zh/docs/agents/#additional)。

## 加载模型

当 opencode 启动时，它会按以下优先级顺序检查模型：

1. `--model` 或 `-m` 命令行标志。格式与配置文件中的格式相同：`provider_id/model_id`。

2. opencode 配置中的模型列表。

   ```json title="opencode.json"
   {
     "$schema": "https://opencode.ai/config.json",
     "model": "anthropic/claude-sonnet-4-20250514"
   }
   ```

   这里的格式是 `provider/model`。

3. 最后使用的模型。

4. 使用内部优先级的第一模型。
