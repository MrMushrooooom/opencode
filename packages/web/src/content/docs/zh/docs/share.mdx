---
title: 分享
description: 分享您的 opencode 对话。
---

opencode 的分享功能允许您为 opencode 对话创建公共链接，这样您就可以与团队成员协作或从其他人那里获得帮助。

:::note
共享对话对任何有链接的人都是公开可访问的。
:::

---

## 工作原理

当您分享对话时，opencode：

1. 为您的会话创建唯一的公共 URL
2. 将您的对话历史同步到我们的服务器
3. 通过可分享的链接使对话可访问 — `opencode.ai/s/<share-id>`

---

## 分享

opencode 支持三种分享模式来控制对话的分享方式：

---

### 手动（默认）

默认情况下，opencode 使用手动分享模式。会话不会自动分享，但您可以使用 `/share` 命令手动分享它们：

```
/share
```

这将生成一个唯一的 URL，该 URL 将被复制到您的剪贴板。

要在您的[配置文件](/docs/config)中明确设置手动模式：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "share": "manual"
}
```

---

### 自动分享

您可以通过在[配置文件](/docs/config)中将 `share` 选项设置为 `"auto"` 来为所有新对话启用自动分享：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "share": "auto"
}
```

启用自动分享后，每个新对话都会自动分享并生成链接。

---

### 禁用

您可以通过在[配置文件](/docs/config)中将 `share` 选项设置为 `"disabled"` 来完全禁用分享：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "share": "disabled"
}
```

要在您的团队中为给定项目强制执行此操作，请将其添加到项目中的 `opencode.json` 并提交到 Git。

---

## 取消分享

要停止分享对话并从公共访问中删除它：

```
/unshare
```

这将删除分享链接并删除与对话相关的数据。

---

## 隐私

分享对话时需要注意一些事项。

---

### 数据保留

共享对话保持可访问状态，直到您明确取消分享。这包括：

- 完整的对话历史
- 所有消息和响应
- 会话元数据

---

### 建议

- 只分享不包含敏感信息的对话。
- 分享前审查对话内容。
- 协作完成后取消分享对话。
- 避免分享包含专有代码或机密数据的对话。
- 对于敏感项目，完全禁用分享。

---

## 企业版

对于企业部署，分享功能可以：

- **完全禁用**以确保安全合规
- **限制**为仅通过 SSO 进行身份验证的用户
- **自托管**在您自己的基础设施上

[了解更多](/docs/enterprise) 关于在您的组织中使用 opencode。
