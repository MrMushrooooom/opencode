---
title: 插件
description: 编写你自己的插件来扩展 opencode。
---

插件允许你通过钩入各种事件和自定义行为来扩展 opencode。你可以创建插件来添加新功能、与外部服务集成或修改 opencode 的默认行为。

---

## 创建插件

插件是一个**JavaScript/TypeScript 模块**，它导出一个或多个插件函数。每个函数接收一个上下文对象并返回一个钩子对象。

---

### 位置

插件从以下位置加载：

1. 你项目中的 `.opencode/plugin` 目录
2. 或者，全局在 `~/.config/opencode/plugin`

---

### 基本结构

```js title=".opencode/plugin/example.js"
export const MyPlugin = async ({ app, client, $ }) => {
  console.log("Plugin initialized!")

  return {
    // Hook implementations go here
  }
}
```

插件函数接收：

- `app`：opencode 应用程序实例。
- `client`：用于与 AI 交互的 opencode SDK 客户端。
- `$`：Bun 的 [shell API](https://bun.com/docs/runtime/shell)，用于执行命令。

---

### TypeScript 支持

对于 TypeScript 插件，你可以从插件包导入类型：

```ts title="my-plugin.ts" {1}
import type { Plugin } from "@opencode-ai/plugin"

export const MyPlugin: Plugin = async ({ app, client, $ }) => {
  return {
    // Type-safe hook implementations
  }
}
```

## 示例

以下是一些你可以用来扩展 opencode 的插件示例。

---

### 发送通知

在特定事件发生时发送通知：

```js title=".opencode/plugin/notification.js"
export const NotificationPlugin = async ({ client, $ }) => {
  return {
    event: async ({ event }) => {
      // 在会话完成时发送通知
      if (event.type === "session.idle") {
        await $`osascript -e 'display notification "Session completed!" with title "opencode"'`
      }
    },
  }
}
```

我们使用 `osascript` 在 macOS 上运行 AppleScript。这里我们用它来发送通知。

---

### .env 保护

防止 opencode 读取 `.env` 文件：

```javascript title=".opencode/plugin/env-protection.js"
export const EnvProtection = async ({ client, $ }) => {
  return {
    "tool.execute.before": async (input, output) => {
      if (input.tool === "read" && output.args.filePath.includes(".env")) {
        throw new Error("Do not read .env files")
      }
    }
  }
}
```
