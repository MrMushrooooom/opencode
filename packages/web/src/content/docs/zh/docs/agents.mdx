title: 代理

---

代理是专门的 AI 助手，可以为特定任务和工作流程进行配置。它们允许你创建具有自定义提示、模型和工具访问权限的专注工具。

## 创建代理

你可以使用 `opencode agent create` 命令创建新代理。这个交互式命令将：

1. 询问在哪里保存代理（全局或项目特定）
2. 提示描述代理应该做什么
3. 生成适当的系统提示和标识符
4. 让你选择代理可以访问哪些工具
5. 创建包含代理配置的 markdown 文件

```bash
opencode agent create
```

该命令将指导你完成整个过程，并根据你的要求自动生成结构良好的代理。

## 内置代理

opencode 带有一个内置的 `general` 代理：

- **general** - 用于研究复杂问题、搜索代码和执行多步骤任务的通用代理。当搜索关键词或文件并且你不确定在前几次尝试中能找到正确匹配时使用此代理。

## 配置

代理可以在 `opencode.json` 配置文件中配置，或作为 markdown 文件配置。

### JSON 配置

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "agent": {
    "code-reviewer": {
      "description": "审查代码的最佳实践和潜在问题",
      "model": "anthropic/claude-sonnet-4-20250514",
      "prompt": "你是一个代码审查员。专注于安全性、性能和可维护性。",
      "tools": {
        "write": false,
        "edit": false
      }
    },
    "test-writer": {
      "description": "用于编写全面测试的专门代理",
      "prompt": "你是一个测试编写专家。编写彻底、可维护的测试。",
      "tools": {
        "bash": true,
        "read": true,
        "write": true
      }
    }
  }
}
```

### Markdown 配置

你也可以使用 markdown 文件定义代理。将它们放在：

- 全局：`~/.config/opencode/agent/`
- 项目：`.opencode/agent/`

```markdown title="~/.config/opencode/agent/code-reviewer.md"
---
description: 审查代码的最佳实践和潜在问题
model: anthropic/claude-sonnet-4-20250514
tools:
  write: false
  edit: false
---

你是一个在安全性、性能和可维护性方面具有专业知识的代码审查员。

专注于：

- 安全漏洞
- 性能瓶颈
- 代码可维护性
- 最佳实践遵循
```

## 代理属性

### 必需

- **description** - 代理做什么以及何时使用它的简要描述

### 可选

- **model** - 要使用的特定模型（默认为你配置的模型）
- **prompt** - 代理的自定义系统提示
- **tools** - 指定代理可以访问哪些工具的对象（每个工具为 true/false）
- **disable** - 设置为 true 以禁用代理

## 工具访问

默认情况下，代理继承与主助手相同的工具访问权限。你可以限制或启用特定工具：

```json
{
  "agent": {
    "readonly-agent": {
      "description": "用于分析的只读代理",
      "tools": {
        "write": false,
        "edit": false,
        "bash": false
      }
    }
  }
}
```

你可能想要控制的常见工具：

- `write` - 创建新文件
- `edit` - 修改现有文件
- `bash` - 执行 shell 命令
- `read` - 读取文件
- `glob` - 搜索文件
- `grep` - 搜索文件内容

## 使用代理

配置后，代理通过任务工具自动可用。主助手将根据它们的描述将它们用于专门任务。

## 最佳实践

1. **清晰的描述** - 编写具体的描述，帮助主助手知道何时使用每个代理
2. **专注的提示** - 保持代理提示专注于它们的特定角色
3. **适当的工具访问** - 只给代理它们任务所需的工具
4. **一致的命名** - 为你的代理使用描述性、一致的名称
5. **项目特定代理** - 对项目特定工作流程使用 `.opencode/agent/`

## 示例

### 文档代理

```markdown title="~/.config/opencode/agent/docs-writer.md"
---
description: 编写和维护项目文档
tools:
  bash: false
---

你是一个技术写手。创建清晰、全面的文档。

专注于：

- 清晰的解释
- 适当的结构
- 代码示例
- 用户友好的语言
```

### 安全审计员

```markdown title="~/.config/opencode/agent/security-auditor.md"
---
description: 执行安全审计并识别漏洞
tools:
  write: false
  edit: false
---

你是一个安全专家。专注于识别潜在的安全问题。

寻找：

- 输入验证漏洞
- 身份验证和授权缺陷
- 数据泄露风险
- 依赖项漏洞
- 配置安全问题
```
