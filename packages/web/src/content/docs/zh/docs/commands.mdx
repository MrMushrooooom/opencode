---

自定义命令允许您指定在 TUI 中执行该命令时要运行的提示。

```bash frame="none"
/my-command
```

自定义命令是对内置命令的补充，如 `/init`、`/undo`、`/redo`、`/share`、`/help`。[了解更多](/docs/tui#commands)。

---

## 创建命令文件

在 `command/` 目录中创建 markdown 文件来定义自定义命令。

创建 `.opencode/command/test.md`：

```md title=".opencode/command/test.md"
---
description: 运行测试并生成覆盖率报告
agent: build
model: anthropic/claude-3-5-sonnet-20241022
---

运行完整的测试套件并生成覆盖率报告，显示任何失败的测试。
重点关注失败的测试并建议修复方案。
```

frontmatter 定义了命令属性。内容成为模板。

通过输入 `/` 后跟命令名称来使用命令。

```bash frame="none"
"/test"
```

---

## 配置

您可以通过 opencode 配置或在 `command/` 目录中创建 markdown 文件来添加自定义命令。

---

### JSON

在您的 opencode [配置](/docs/config) 中使用 `command` 选项：

```json title="opencode.jsonc" {4-12}
{
  "$schema": "https://opencode.ai/config.json",
  "command": {
    // 这成为命令的名称
    "test": {
      // 这是将发送给 LLM 的提示
      "template": "运行完整的测试套件并生成覆盖率报告，显示任何失败的测试。\n重点关注失败的测试并建议修复方案。",
      // 这在 TUI 中显示为描述
      "description": "运行测试并生成覆盖率报告",
      "agent": "build",
      "model": "anthropic/claude-3-5-sonnet-20241022"
    },
  }
}
```

现在您可以在 TUI 中运行此命令：

```bash frame="none"
/test
```

---

### Markdown

您也可以使用 markdown 文件定义命令。将它们放在：

- 全局：`~/.config/opencode/command/`
- 每个项目：`.opencode/command/`

```markdown title="~/.config/opencode/command/test.md"
---
description: 运行测试并生成覆盖率报告
agent: build
model: anthropic/claude-3-5-sonnet-20241022
---

运行完整的测试套件并生成覆盖率报告，显示任何失败的测试。
重点关注失败的测试并建议修复方案。
```

markdown 文件名成为命令名称。例如，`test.md` 允许您运行：

```bash frame="none"
/test
```
