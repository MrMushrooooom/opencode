[**opencode**](/) 是一个为终端而构建的 AI 编程助手。

![带有 opencode 主题的 opencode TUI](/src/assets/lander/screenshot.png)

让我们开始吧。

---

#### 前置要求

要使用 opencode，您需要：

1. 现代终端模拟器，例如：
   - [WezTerm](https://wezterm.org)，跨平台
   - [Alacritty](https://alacritty.org)，跨平台
   - [Ghostty](https://ghostty.org)，Linux 和 macOS
   - [Kitty](https://sw.kovidgoyal.net/kitty/)，Linux 和 macOS

2. 您想使用的大语言模型提供商的 API 密钥。

---

## 安装

安装 opencode 最简单的方法是使用安装脚本。

```bash
curl -fsSL https://opencode.ai/install | bash
```

您也可以通过以下方式安装：

- **使用 Node.js**

  <Tabs>
    <TabItem label="npm">```bash npm install -g opencode-ai ```</TabItem>
    <TabItem label="Bun">```bash bun install -g opencode-ai ```</TabItem>
    <TabItem label="pnpm">```bash pnpm install -g opencode-ai ```</TabItem>
    <TabItem label="Yarn">```bash yarn global add opencode-ai ```</TabItem>
  </Tabs>

- **在 macOS 和 Linux 上使用 Homebrew**

  ```bash
  brew install sst/tap/opencode
  ```

- **在 Arch Linux 上使用 Paru**

  ```bash
  paru -S opencode-bin
  ```

#### Windows

目前自动安装方法在 Windows 上无法正常工作。但是您可以从 [Releases](https://github.com/sst/opencode/releases) 获取二进制文件。

## 配置

使用 opencode，您可以通过配置 API 密钥来使用任何大语言模型提供商。

我们推荐注册 [Claude Pro](https://www.anthropic.com/news/claude-pro) 或 [Max](https://www.anthropic.com/max)，这是使用 opencode 最具性价比的方式。

注册完成后，运行 `opencode auth login` 并选择 Anthropic。

```bash
$ opencode auth login

┌  添加凭证
│
◆  选择提供商
│  ● Anthropic（推荐）
│  ○ OpenAI
│  ○ Google
│  ○ Amazon Bedrock
│  ○ Azure
│  ○ DeepSeek
│  ○ Groq
│  ...
└
```

或者，您可以选择其他提供商之一。[了解更多](/docs/providers#directory)。

## 初始化

现在您已经配置了提供商，可以导航到您想要工作的项目。

```bash
cd /path/to/project
```

并运行 opencode。

```bash
opencode
```

接下来，通过运行以下命令为项目初始化 opencode。

```bash frame="none"
/init
```

这将让 opencode 分析您的项目并在项目根目录中创建 `AGENTS.md` 文件。

:::tip
您应该将项目的 `AGENTS.md` 文件提交到 Git。
:::

这有助于 opencode 理解项目结构和使用的编码模式。

## 使用方法

您现在可以使用 opencode 来处理您的项目了。随时询问它任何问题！

如果您是 AI 编程助手的新用户，以下是一些可能有帮助的示例。

---

### 提出问题

您可以要求 opencode 为您解释代码库。

:::tip
使用 `@` 键在项目中模糊搜索文件。
:::

```txt frame="none" "@packages/functions/src/api/index.ts"
@packages/functions/src/api/index.ts 中的身份验证是如何处理的？
```

如果有您没有参与开发的代码库部分，这会很有帮助。

---

### 添加功能

您可以要求 opencode 为您的项目添加新功能。不过我们首先建议您要求它创建一个计划。

1. **创建计划**

   opencode 有一个 _计划模式_，该模式会禁用其更改能力，而是建议它将 _如何_ 实现该功能。

   使用 **Tab** 键切换到该模式。您会在右下角看到此模式的指示器。

   ```bash frame="none" title="切换到计划模式"
   <TAB>
   ```

   现在让我们描述我们希望它做什么。

   ```txt frame="none"
   当用户删除笔记时，我们希望在数据库中将其标记为已删除。
   然后创建一个显示所有最近删除笔记的界面。
   在此界面中，用户可以恢复删除笔记或永久删除它。
   ```

   您要给 opencode 足够的细节来理解您想要什么。这就像您在与团队中的初级开发人员交谈一样。

   :::tip
   给 opencode 提供大量上下文和示例，以帮助它理解您想要什么。
   :::

2. **迭代计划**

   一旦它给您一个计划，您可以提供反馈或添加更多细节。

   ```txt frame="none"
   我们希望使用我之前用过的设计来设计这个新界面。
   [Image #1] 看看这张图片并将其用作参考。
   ```

   :::tip
   将图片拖放到终端中以将其添加到提示中。
   :::

   opencode 可以扫描您提供的任何图片并将其添加到提示中。您可以通过将图片拖放到终端中来实现此操作。

3. **构建功能**

   一旦您对计划感到满意，通过再次按 **Tab** 键切换回 _构建模式_。

   ```bash frame="none"
   <TAB>
   ```

   并要求它进行更改。

   ```bash frame="none"
   听起来不错！继续进行更改吧。
   ```

---

### 进行更改

对于更直接的更改，您可以要求 opencode 直接构建，而无需首先查看计划。

```txt frame="none" "@packages/functions/src/settings.ts" "@packages/functions/src/notes.ts"
我们需要为 /settings 路由添加身份验证。看看这在 @packages/functions/src/notes.ts 中的 /notes 路由中是如何处理的，并在 @packages/functions/src/settings.ts 中实现相同的逻辑。
```

您要确保提供大量细节，以便 opencode 做出正确的更改。

---

### 撤销更改

假设您要求 opencode 进行一些更改。

```txt frame="none" "@packages/functions/src/api/index.ts"
您能重构 @packages/functions/src/api/index.ts 中的函数吗？
```

但您意识到这不是您想要的。您 **可以撤销** 使用 `/undo` 命令的更改。

```bash frame="none"
/undo
```

opencode 现在将恢复您所做的更改并再次显示您的原始消息。

```txt frame="none" "@packages/functions/src/api/index.ts"
您能重构 @packages/functions/src/api/index.ts 中的函数吗？
```

从这里您可以调整提示并要求 opencode 再次尝试。

:::tip
您可以多次运行 `/undo` 来撤销多个更改。
:::

或者您 **可以重做** 使用 `/redo` 命令的更改。

```bash frame="none"
/redo
```

## 分享

您与 opencode 的对话可以[与您的团队分享](/docs/share)。

```bash frame="none"
/share
```

这将创建当前对话的链接并将其复制到您的剪贴板。

:::note
对话默认不会被分享。
:::
