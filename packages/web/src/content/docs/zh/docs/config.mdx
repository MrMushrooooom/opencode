你可以使用 JSON 配置文件来配置 opencode。

## 格式

opencode 支持 JSON 和 JSONC（带注释的 JSON）格式。你可以在配置文件中使用注释：

```jsonc title="opencode.jsonc"
{
  "$schema": "https://opencode.ai/config.json",
  // 主题配置
  "theme": "opencode",
  "model": "anthropic/claude-sonnet-4-20250514",
  "autoupdate": true,
}
```

这可以用于全局配置 opencode 或为特定项目配置。

---

### 全局配置

将你的全局 opencode 配置放在 `~/.config/opencode/opencode.json` 中。你会想要使用全局配置来设置主题、提供商或快捷键等。

---

### 项目级配置

你也可以在项目中添加 `opencode.json`。这对于配置特定于你项目的提供商或模式很有用。

当 opencode 启动时，它会在当前目录中查找配置文件，或者向上遍历到最近的 Git 目录。

这个文件也可以安全地提交到 Git，并使用与全局配置相同的模式。

---

### 自定义配置文件

你可以使用 `OPENCODE_CONFIG` 环境变量指定自定义配置文件。这会优先于全局和项目配置。

```bash
export OPENCODE_CONFIG=/path/to/my/custom-config.json
opencode run "Hello world"
```

---

## 模式

配置文件有一个在 [**`opencode.ai/config.json`**](https://opencode.ai/config.json) 中定义的模式。

你的编辑器应该能够基于模式进行验证和自动完成。

---

### 模式

opencode 带有两个内置模式：_build_，默认启用所有工具。和 _plan_，限制模式，禁用文件修改工具。你可以覆盖这些内置模式或使用 `mode` 选项定义自己的自定义模式。

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "mode": {
    "build": {},
    "plan": {},
    "my-custom-mode": {}
  }
}
```

[了解更多](/zh/docs/modes)。

---

### 模型

你可以通过 `provider`、`model` 和 `small_model` 选项在 opencode 配置中配置你想要使用的提供商和模型。

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {},
  "model": "anthropic/claude-sonnet-4-20250514",
  "small_model": "anthropic/claude-3-5-haiku-20241022"
}
```

`small_model` 选项配置一个单独的模型用于轻量级任务，如摘要和标题生成。默认情况下，opencode 会尝试使用更便宜的模型（如果提供商有的话），否则会回退到你的主模型。

你也可以配置[本地模型](/zh/docs/models#local)。[了解更多](/zh/docs/models)。

---

### 主题

你可以通过 `theme` 选项在 opencode 配置中配置你想要使用的主题。

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "theme": ""
}
```

[了解更多](/zh/docs/themes)。

---

### 日志

日志写入到：

- **macOS/Linux**：`~/.local/share/opencode/log/`
- **Windows**：`%APPDATA%\opencode\log\`

---

### 分享

你可以通过 `share` 选项配置[分享](/zh/docs/share)功能。

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "share": "manual"
}
```

这接受：

- `"manual"` - 允许通过命令手动分享（默认）
- `"auto"` - 自动分享新对话
- `"disabled"` - 完全禁用分享

默认情况下，分享设置为手动模式，你需要使用 `/share` 命令明确分享对话。

---

### 快捷键

你可以通过 `keybinds` 选项自定义快捷键。

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "keybinds": {}
}
```

[了解更多](/zh/docs/keybinds)。

---

### 自动更新

opencode 会在启动时自动下载任何新更新。你可以使用 `autoupdate` 选项禁用此功能。

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "autoupdate": false
}
```

---

### MCP 服务器

你可以通过 `mcp` 选项配置你想要使用的 MCP 服务器。

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {}
}
```

[了解更多](/zh/docs/mcp-servers)。

---

### 指令

你可以通过 `instructions` 选项配置你正在使用的模型的指令。

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "instructions": ["CONTRIBUTING.md", "docs/guidelines.md", ".cursor/rules/*.md"]
}
```

这接受一个路径和 glob 模式数组来指定指令文件。[了解更多关于规则](/zh/docs/rules)。

---

### 代理

你可以通过 `agent` 选项为特定任务配置专门的代理。

```jsonc title="opencode.jsonc"
{
  "$schema": "https://opencode.ai/config.json",
  "agent": {
    "code-reviewer": {
      "description": "审查代码的最佳实践和潜在问题",
      "model": "anthropic/claude-sonnet-4-20250514",
      "prompt": "你是一个代码审查员。专注于安全性、性能和可维护性。",
      "tools": {
        // 为仅审查代理禁用文件修改工具
        "write": false,
        "edit": false,
      },
    },
  },
}
```

你也可以使用 `~/.config/opencode/agent/` 或 `.opencode/agent/` 中的 markdown 文件定义代理。[了解更多](/zh/docs/agents)。

---

### 禁用的提供商

你可以通过 `disabled_providers` 选项禁用自动加载的提供商。当你想要防止某些提供商被加载，即使它们的凭据可用时，这很有用。

---

### 格式化器

你可以通过 `formatter` 选项配置代码格式化器。查看[格式化器文档](/zh/docs/formatters)了解更多详情。

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "disabled_providers": ["openai", "gemini"]
}
```

`disabled_providers` 选项接受提供商 ID 数组。当提供商被禁用时：

- 即使设置了环境变量也不会被加载
- 即使通过 `opencode auth login` 配置了 API 密钥也不会被加载
- 提供商的模型不会出现在模型选择列表中

---

### 权限

你可以通过 `permission` 选项配置权限来控制 AI 代理在你的代码库中可以做什么。

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "permission": {
    "edit": "ask",
    "bash": "ask"
  }
}
```

权限系统允许你为敏感操作配置明确的批准要求：

- `edit` - 控制文件编辑操作是否需要用户批准（`"ask"` 或 `"allow"`）
- `bash` - 控制 bash 命令是否需要用户批准（可以是 `"ask"`/`"allow"` 或模式映射）

[了解更多关于权限](/zh/docs/permissions)。

## 变量

你可以在配置文件中使用变量替换来引用环境变量和文件内容。

---

### 环境变量

使用 `{env:VARIABLE_NAME}` 来替换环境变量：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "model": "{env:OPENCODE_MODEL}",
  "provider": {
    "anthropic": {
      "options": {
        "apiKey": "{env:ANTHROPIC_API_KEY}"
      }
    }
  }
}
```

如果环境变量未设置，它将被替换为空字符串。

---

### 文件

使用 `{file:path/to/file}` 来替换文件内容：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "instructions": ["{file:./custom-instructions.md}"],
  "provider": {
    "openai": {
      "options": {
        "apiKey": "{file:~/.secrets/openai-key}"
      }
    }
  }
}
```

文件路径可以是：

- 相对于配置文件目录
- 或者以 `/` 或 `~` 开头的绝对路径

这些对于以下情况很有用：

- 将敏感数据（如 API 密钥）保存在单独的文件中。
- 包含大型指令文件而不使配置混乱。
- 在多个配置文件中共享通用配置片段。
