# OpenCode 配置指南，配置 opencode 的最佳方式

欢迎使用 OpenCode 的全面配置指南！本文档涵盖了配置 OpenCode 环境以实现最佳性能和自定义化的所有方面。

> **注意**：这是一份全面的配置指南，涵盖了 OpenCode 中所有主要功能和选项。

---

## 快速开始

要开始配置 OpenCode，请在项目根目录中创建 `opencode.json` 文件：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "theme": "opencode",
  "model": "anthropic/claude-sonnet-4-20250514",
  "autoupdate": true
}
```

---

## 核心配置选项

### 基本设置

OpenCode 最重要的配置选项：

| 选项         | 类型    | 默认值                                 | 描述                 |
| ------------ | ------- | -------------------------------------- | -------------------- |
| `theme`      | string  | `"opencode"`                           | 界面视觉主题         |
| `model`      | string  | `"anthropic/claude-sonnet-4-20250514"` | 要使用的主要 AI 模型 |
| `autoupdate` | boolean | `true`                                 | 启用自动更新         |

### 高级设置

针对需要更多控制权限的高级用户：

```jsonc title="opencode.jsonc"
{
  "$schema": "https://opencode.ai/config.json",
  "theme": "opencode",
  "model": "anthropic/claude-sonnet-4-20250514",
  "small_model": "anthropic/claude-3-5-haiku-20241022",
  "autoupdate": true,
  "share": "manual",
  "permission": {
    "edit": "ask",
    "bash": "ask",
  },
}
```

## 模型配置

### 提供商设置

配置不同的 AI 提供商及其模型：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "anthropic": {
      "models": {
        "claude-sonnet-4": "anthropic/claude-sonnet-4-20250514",
        "claude-haiku": "anthropic/claude-3-5-haiku-20241022"
      },
      "options": {
        "apiKey": "{env:ANTHROPIC_API_KEY}"
      }
    },
    "openai": {
      "models": {
        "gpt-4": "openai/gpt-4",
        "gpt-3.5": "openai/gpt-3.5-turbo"
      },
      "options": {
        "apiKey": "{env:OPENAI_API_KEY}"
      }
    }
  }
}
```

### 模型选择

为不同任务选择合适的模型：

- **Claude Sonnet 4**：最适合复杂推理和代码生成
- **Claude Haiku**：快速且经济高效，适合简单任务
- **GPT-4**：非常适合创意写作和分析
- **GPT-3.5**：速度和能力的良好平衡

## 智能体配置

### 内置智能体

OpenCode 附带多个预配置的智能体：

```jsonc title="opencode.jsonc"
{
  "$schema": "https://opencode.ai/config.json",
  "agent": {
    "build": {
      "description": "用于开发任务的主要智能体",
      "mode": "primary",
      "model": "anthropic/claude-sonnet-4-20250514",
      "tools": {
        "write": true,
        "edit": true,
        "bash": true,
      },
    },
    "plan": {
      "description": "规划和分析智能体",
      "mode": "primary",
      "model": "anthropic/claude-haiku-4-20250514",
      "tools": {
        "write": false,
        "edit": false,
        "bash": false,
      },
    },
    "code-reviewer": {
      "description": "专门的代码审查智能体",
      "mode": "subagent",
      "model": "anthropic/claude-sonnet-4-20250514",
      "prompt": "You are a code reviewer. Focus on security, performance, and maintainability.",
      "tools": {
        "write": false,
        "edit": false,
      },
    },
  },
}
```

### 自定义智能体创建

创建您自己的专门智能体：

```markdown title="~/.config/opencode/agent/custom.md"
---
description: 针对特定项目需求的自定义智能体
mode: subagent
model: anthropic/claude-sonnet-4-20250514
temperature: 0.1
tools:
  write: true
  edit: true
  bash: false
---

您是一个专门从事 [您的特定领域] 的自定义智能体。
专注于 [特定任务或专业领域]。
```

## 主题配置

### 可用主题

OpenCode 支持多种视觉主题：

- **opencode**：具有现代设计的默认主题
- **dark**：适合低光环境的暗色模式
- **light**：适合明亮环境的浅色模式
- **custom**：用户定义的自定义主题

### 自定义主题创建

```json title="~/.config/opencode/themes/custom.json"
{
  "name": "自定义主题",
  "description": "我的个性化 OpenCode 主题",
  "colors": {
    "primary": "#007acc",
    "secondary": "#6c757d",
    "background": "#ffffff",
    "text": "#212529"
  },
  "fonts": {
    "monospace": "Fira Code",
    "sans": "Inter"
  }
}
```

## 权限系统

### 安全级别

配置 AI 智能体在您的环境中可以执行的操作：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "permission": {
    "edit": "ask",
    "bash": {
      "*": "ask",
      "safe-commands": "allow"
    },
    "webfetch": "allow"
  }
}
```

### 权限选项

- **`"allow"`**：无需确认
- **`"ask"`**：需要用户确认
- **`"deny"`**：阻止操作

## 格式化程序配置

### 内置格式化程序

配置代码格式化工具：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "formatter": {
    "prettier": {
      "disabled": false,
      "config": ".prettierrc"
    },
    "eslint": {
      "disabled": false,
      "fix": true
    },
    "black": {
      "disabled": false,
      "line_length": 88
    }
  }
}
```

### 自定义格式化程序

添加您自己的格式化命令：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "formatter": {
    "custom-prettier": {
      "command": ["npx", "prettier", "--write", "$FILE"],
      "environment": {
        "NODE_ENV": "development"
      },
      "extensions": [".js", ".ts", ".jsx", ".tsx"]
    }
  }
}
```

## MCP 服务器配置

### 模型上下文协议

配置 MCP 服务器以增强 AI 能力：

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "servers": {
      "github": {
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-github"],
        "env": {
          "GITHUB_TOKEN": "{env:GITHUB_TOKEN}"
        }
      },
      "filesystem": {
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-filesystem"]
      }
    }
  }
}
```

## 最佳实践

### 配置组织

1. **全局设置**：在 `~/.config/opencode/opencode.json` 中放置通用设置
2. **项目设置**：使用项目特定的 `opencode.json` 覆盖设置
3. **环境变量**：用于 API 密钥等敏感数据
4. **文件引用**：包含大型指令文件而不会使配置混乱

### 安全考虑

- 永远不要直接将 API 密钥提交到版本控制中
- 使用环境变量处理敏感配置
- 在允许 AI 操作之前仔细审查权限
- 首先在安全环境中测试配置

## 故障排除

### 常见问题

| 问题         | 解决方案                  |
| ------------ | ------------------------- |
| 配置未加载   | 检查文件路径和 JSON 语法  |
| 模型不可用   | 验证提供商配置和 API 密钥 |
| 权限过于严格 | 在配置中调整权限级别      |
| 主题未应用   | 确保主题文件存在且有效    |

### 调试模式

启用调试日志：

```bash
export OPENCODE_DEBUG=1
opencode run "test command"
```
