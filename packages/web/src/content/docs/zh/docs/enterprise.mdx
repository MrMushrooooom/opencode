## 试用

由于 opencode 是开源的，不存储你的任何代码或上下文数据，你的开发人员可以简单地[开始使用](/docs/)并进行试用。

---

### 数据处理

**opencode 不存储你的代码或上下文数据。** 所有处理都在本地进行或通过直接 API 调用到你的 AI 提供商。

这里唯一的例外是可选的 `/share` 功能。

---

#### 分享对话

如果用户启用 `/share` 功能，对话和与之相关的数据会发送到我们用于托管 opencode.ai 这些分享页面的服务。

数据目前通过我们的 CDN 边缘网络提供服务，并缓存在用户附近的边缘。

我们建议你在试用期间禁用此功能。

```json title="opencode.json"
{
  "$schema": "https://opencode.ai/config.json",
  "share": "disabled"
}
```

[了解更多关于分享](/docs/share)。

---

### 代码所有权

**你拥有 opencode 产生的所有代码。** 没有许可限制或所有权声明。

### 私有 NPM

opencode 通过 Bun 的原生 `.npmrc` 文件支持支持私有 npm 注册表。如果你的组织使用私有注册表，如 JFrog Artifactory、Nexus 或类似产品，确保开发人员在运行 opencode 之前已通过认证。

要设置与私有注册表的认证：

```bash
npm login --registry=https://your-company.jfrog.io/api/npm/npm-virtual/
```

这会创建包含认证详情的 `~/.npmrc`。opencode 会自动获取这些信息。

:::caution
在运行 opencode 之前，你必须登录到私有注册表。
:::

或者，你可以手动配置 `.npmrc` 文件：

```bash title="~/.npmrc"
registry=https://your-company.jfrog.io/api/npm/npm-virtual/
//your-company.jfrog.io/api/npm/npm-virtual/:_authToken=${NPM_AUTH_TOKEN}
```

开发人员必须在运行 opencode 之前登录到私有注册表，以确保可以从企业注册表安装包。
